## 5. 핑퐁의 진짜 깨달음 3개

### A. 입장 바뀐 항목 — "비용 0이라는 인정의 무게"
v1에서 DEFER했던 발음 함정 표시(2.5)/표제어 TTS(1.3)/응답 시간 측정(1.10/1.11/3.6) 모두 v2/v3에서 "자동 추출 가능 / 데이터 비용 0 / 인프라 존재"임을 인정한 순간 v1.0 격상. **PM 입장에서 가장 자주 빠지는 함정**: "비용 작은 일을 작다고 인정하면서도 'v1.x'라는 소속만 부여해 미루기". 핑퐁이 이걸 강제로 깸. 결과: F9, F16, F17 모두 v1.0 출시 차단으로 들어옴.

### B. 끝까지 평행선이던 항목 — 없음 (DEADLOCK 0)
3라운드를 돌렸는데 진짜 평행선은 0건. 5.2(와우 모먼트)와 3.9(찍기 신뢰도)가 v1→v2 HOLD로 갔지만 v3에서 한쪽이 양보 (5.2는 "한국 시장에 4지선다 앱 이미 많다"는 구체 사실로 무너짐, 3.9는 마사키 자체 WITHDRAW). **시사점**: 진짜 평행선은 사실(시장/데이터/제약)이 부족할 때 발생. 사실이 들어오면 한쪽이 자연스레 무너진다. 이번 핑퐁이 그 사실을 끌어냈다.

### C. 양쪽 다 놓친 사각지대 — **"베타 데이터 회수 절차"**
v1/v2 모두 "v1.0 = 베타로 좁히면 리텐션 차단 면제"로 합의했지만, 마사키가 v3에서 정조준: **"베타에서 D7 측정한다며? 외부 송신 없는데 어떻게 회수하나?"** (5.5). 답: 수동 export + 사용자가 본인에게 파일 전송 (L1과 같은 인프라). v1/v2 동안 양쪽 다 "측정만 하면 됨"으로 넘어갔던 부분. **베타 단계에서 가장 약한 고리**: 외부 SDK 거부 + 자동 회수 부재 → 측정해도 데이터 누적 안 됨. F13 JSON export 인프라가 회수 채널 겸용.

## 6. 교착 항목 (DEADLOCK)

**없음**. 3라운드 핑퐁에서 모든 HOLD 항목이 한쪽 양보 또는 마사키 WITHDRAW로 해소. 사람이 직접 봐야 할 부분은 4번 ESCALATE 3개로 모두 좁혀짐.
