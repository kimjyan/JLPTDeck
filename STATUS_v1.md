## 4. 알려진 문제점 (PM 관점)

### High
1. **한국어 번역 품질 미검증** — 7,316개 전부 AI 단발 생성, 사람 리뷰 0건.
   다의어/관용 표현/뉘앙스 오류 가능성. 4지선다라 **틀린 정답**이 섞이면 학습 손상 직접적.
2. **라이선스 미정리** — LICENSE 파일 부재 + Tanos 라이선스 불명 + 앱 내 attribution UI 미확인.
   Apple 심사 또는 상업 배포 시 차단 위험.
3. **외부 리텐션 트리거 0건** — 푸시/위젯/Live Activity 모두 없음. 사용자가 앱을 "잊으면" 끝.
   Streak 카운터는 앱을 켰을 때만 보임 (intrinsic only).

### Medium
4. **Recognition만, Production 없음** — 4지선다는 보고 고르기. "한국어 → 일본어 떠올리기"
   (생성형 회상)는 학습 효과가 더 큰데 미구현.
5. **Distractor 품질 불명** — `QuizGenerator`는 distractor pool에서 랜덤 픽. 의미적으로
   너무 가깝거나(헷갈림 과도) 너무 멀어(찍기 쉬움) 학습 효과 들쭉날쭉할 수 있음. 측정 안 함.
6. **SwiftData 테스트 크래시** (CLAUDE.md 명시) — `JMdictImporterTests`,
   `LocalRepositoryTests` 등 5개 테스트가 host-app deinit 크래시로 비활성. 운영 환경에서도
   같은 코드 경로가 트리거될 가능성 미검증.
7. **첫 실행 자동 import 실패 시 UX** — `HomeView`가 자동 import 트리거. 실패 시 사용자가
   할 수 있는 복구 액션이 명확한지 미확인.

### Low
8. **통계 탭 깊이 부족** (커밋만 봐서) — 진도/카드 수 외에 정답률/주차별 학습량/취약 어휘 등
   행동 변화로 이어지는 인사이트가 있는지 미확인.
9. **N1 학습자 = 일본어 이미 잘함** — 그런 사용자에게 한국어 4지선다가 매력적인지 의문.
   영어 4지선다 옵션이 있어도 좋을 수 있음 (현재는 한국어 고정).

## 6. Anki + JLPT 덱 대비 차별점

1. **즉시 시작 (Zero setup)** — Anki는 덱 검색/임포트/스케줄 설정/카드 템플릿 학습 필요.
   JLPTDeck은 다운로드 → 한 탭 → 학습 시작.
2. **한국어 뜻 + 한국어 UI** — 영어권 JLPT 덱은 흔하나, 한국어 뜻 + 한국어 UI까지 갖춘
   pre-built 7,316단어 덱은 희소. (단, 번역 품질 미검증 — 4번 문제점 참고)
3. **세션 내 오답 즉시 재노출 + 전용 "틀린 단어" 탭** — Anki도 filtered deck으로 가능하나
   여러 단계 설정 필요. JLPTDeck은 기본값으로 자동 + 별도 탭 1탭 진입.

(차별점 외 단점: Anki는 카드 커스터마이즈/오디오/예문/태그/통계가 압도적으로 풍부.
JLPTDeck의 가치 명제는 "단순함 + 한국어"에 한정.)
